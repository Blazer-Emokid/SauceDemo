# Чек-лист тестирования: Swag Labs (https://www.saucedemo.com)

## 1. Введение

**Цель чек-листа:** Проверить основные функции интернет-магазина "Swag Labs", включая авторизацию, каталог, корзину, оформление заказа и наличие визуальных/функциональных дефектов.

**Информация о тестовом стенде:**
- **URL:** [https://www.saucedemo.com/](https://www.saucedemo.com/)
- **Тестовые учетные данные:**
    - **Логины:** `standard_user`, `locked_out_user`, `problem_user`, `performance_glitch_user`, `error_user`, `visual_user`
    - **Пароль для всех:** `secret_sauce`

---

## 2. Чек-лист

### 2.1. Авторизация (Login)

- [ ] **TC-01:** Авторизация с `standard_user` / `secret_sauce` -> успешный вход на главную страницу (`/inventory.html`).
- [ ] **TC-02:** Авторизация с `locked_out_user` / `secret_sauce` -> отображается сообщение об ошибке: *"Epic sadface: Sorry, this user has been locked out."*
- [ ] **TC-03:** Авторизация с неверным паролем (например, `wrong_pass`) -> сообщение: *"Epic sadface: Username and password do not match any user in this service"*.
- [ ] **TC-04:** Авторизация с пустыми полями "Username" и "Password" -> сообщение: *"Epic sadface: Username is required"*.
- [ ] **TC-05:** Выход из системы (клик по бургер-меню -> `Logout`) -> переход на страницу логина, невозможность вернуться в каталог без повторной авторизации.

### 2.2. Каталог товаров (Inventory)

- [ ] **TC-06:** На странице отображаются карточки всех 6 товаров (изображение, название, описание, цена, кнопка "Add to cart").
- [ ] **TC-07:** Изображения товаров загружаются корректно (для `visual_user` проверить визуальные отличия).
- [ ] **TC-08:** Добавление товара в корзину -> кнопка меняется на "Remove", в иконке корзины обновляется счетчик.
- [ ] **TC-09:** Удаление товара из корзины (кнопка "Remove") -> кнопка возвращается к "Add to cart", счетчик корзины уменьшается.
- [ ] **TC-10:** Сортировка "Name (A to Z)" -> товары расположены по алфавиту (от A к Z).
- [ ] **TC-11:** Сортировка "Name (Z to A)" -> товары расположены по обратному алфавиту (от Z к A).
- [ ] **TC-12:** Сортировка "Price (low to high)" -> цены товаров идут от меньшей к большей.
- [ ] **TC-13:** Сортировка "Price (high to low)" -> цены товаров идут от большей к меньшей.
- [ ] **TC-14:** *(Специфично для `problem_user`)*: При добавлении товаров или сортировке наблюдаются неожиданные поведения (например, дублирование изображений, обновление не всех элементов).

### 2.3. Корзина (Cart)

- [ ] **TC-15:** Добавление нескольких товаров в корзину -> в корзине отображаются все добавленные товары с корректными данными.
- [ ] **TC-16:** В корзине можно удалить товар (кнопка "Remove") -> товар исчезает, суммарная стоимость пересчитывается.
- [ ] **TC-17:** Из корзины можно продолжить покупки (кнопка "Continue Shopping") -> возврат в каталог, корзина не сбрасывается.
- [ ] **TC-18:** Переход к оформлению заказа (кнопка "Checkout") -> открывается страница `/checkout-step-one.html`.

### 2.4. Оформление заказа (Checkout)

- [ ] **TC-19:** Отправка пустой формы "Checkout: Your Information" -> сообщение: *"Error: First Name is required"*.
- [ ] **TC-20:** Заполнение только имени и фамилии (без почтового индекса) -> сообщение: *"Error: Postal Code is required"*.
- [ ] **TC-21:** Успешное заполнение всех полей (First Name, Last Name, Zip/Postal Code) -> переход на страницу `/checkout-step-two.html`.
- [ ] **TC-22:** На странице обзора заказа (`Step Two`) отображаются: список товаров, общая сумма (`Item total`), налог (`Tax`) и итоговая цена (`Total`).
- [ ] **TC-23:** Нажатие "Cancel" на любом этапе оформления возвращает в корзину.
- [ ] **TC-24:** Нажатие "Finish" -> завершение заказа, отображение страницы "Thank you for your order!" и сообщения об успешной покупке.

### 2.5. Граничные случаи

- [ ] **TC-25:** Добавление товара в корзину, разлогинивание и логин снова -> корзина должна быть пуста (или сохранена? зависит от требований, ожидаемо — пуста, т.к. данные сессии сброшены).
- [ ] **TC-26:** Использование кнопок браузера "Назад" / "Вперед" на разных этапах оформления заказа (не должно ломать состояние приложения).
- [ ] **TC-27:** Проверка работы бургер-меню: пункты "All Items", "About", "Logout" ведут на корректные страницы.
- [ ] **TC-28:** Валидация ссылки "About" (ведёт на внешний сайт Sauce Labs).
- [ ] **TC-29:** `performance_glitch_user`: измерение времени загрузки страниц (логин, каталог, корзина). [ ] Должно быть заметно дольше, но не приводить к критическим таймаутам.
- [ ] **TC-30:** `error_user`: проверить, что при попытке сортировки или добавления в корзину некоторых товаров возникают ошибки (отсутствие изображения, проблемы с кнопкой).
